Laporan Implementasi dan Konfigurasi Systemd Service

Langkah 1: Membuat File Service

1. Lokasi File Service

File service dibuat di direktori /etc/systemd/system/ dengan nama app.service. Perintah yang digunakan:

sudo nano /etc/systemd/system/app.service

2. Konfigurasi File Service

Isi file service diatur sebagai berikut, disesuaikan dengan path proyek aplikasi Python:

[Unit]
Description=My Python App
After=network.target

[Service]
User=root
WorkingDirectory=/home/ai
ExecStart=/home/ai/venv/bin/python /home/ai/app.py
Restart=always
Environment=PYTHONUNBUFFERED=1

[Install]
WantedBy=multi-user.target

Langkah 2: Reload dan Aktifkan Service

1. Reload Konfigurasi Systemd

sudo systemctl daemon-reload

2. Aktifkan Service Saat Boot

sudo systemctl enable app.service

3. Memulai Service

sudo systemctl start app.service

4. Memeriksa Status Service

sudo systemctl status app.service

Hasil Akhir

  • File service telah dibuat dengan konfigurasi yang sesuai.
  • Service berhasil diaktifkan dan berjalan otomatis saat sistem dihidupkan.
  • Debugging dilakukan jika terdapat error, dan solusi diterapkan berdasarkan log systemd.

Kesimpulan

Implementasi systemd service untuk aplikasi Python telah selesai dilakukan, dan service berhasil berjalan secara otomatis serta dapat diandalkan untuk memastikan aplikasi selalu aktif di server.
